# `planning-beats-loop.jsonl` record schema

One JSON object per line. Append-only. The autonomous loop reads the
full file at iteration N+1 to let Codex propose the next config.

## Fields

```jsonc
{
  // Globally unique; convention: "<sub-loop>-loop-v1-iter-<zero-padded-N>"
  "iteration_id": "planning-beats-loop-v1-iter-003",

  // Constrained to Phase 0 scope. When Sub-loop 2 opens, add "beat-context".
  "sub_loop": "planning-beats",

  // ISO 8601 UTC.
  "proposed_at": "2026-04-21T18:00:00Z",

  // Optional: 2-3 sentence quote from Codex explaining the hypothesis.
  "proposer_reasoning": "…",

  // The knob values Codex proposed. Exactly-one-knob-changed per
  // iteration unless proposer_reasoning pre-registers a 2-knob delta
  // and explains why.
  "config": {
    "richness_tier": "standard | rich | compact",
    "established_facts_target": "0 | 1-2 | 3-5",
    "knowledge_changes_explicitness": "implicit | named-character | named-and-reason",
    "payoff_link_depth": "0 | 1-hop | 2-hop",
    "beat_count_floor_multiplier": 1.0,
    "temperature": 0.6,
    "max_tokens": 8192
    // See docs/archive/2026-04/harness-optimization-inventory.md §1.2 for the full
    // knob list. Phase 0 opens a subset marked Y; later sub-loops add more.
  },

  // Measurement pool size. Frozen at 20 for Phase 0.
  "beats_scored": 20,

  // Sub-loop-specific metrics per the design doc §Sub-loop-1.
  "scores": {
    "beat_count_floor_pass_rate": 1.00,
    "chapter_plan_checker_pass_rate": 0.85,
    "downstream_adherence_pass_rate": 0.82,

    "character_distinctness_pass_rate": null,   // optional: expensive, OFF by default per autonomous-context-loop.md §4 cost note
    "defect_fire_rate": 0.05,
    "token_cost_per_beat": 0.00035,
    "total_cost": 0.47
  },

  // Mandatory for any candidate winner. See composition rule in the
  // design doc. If the upstream change doesn't survive downstream
  // replay, gate_status is "fail" and we don't ship.
  "downstream_replay": {
    "ran": true,
    "frozen_downstream_sha": "<commit-sha>",
    "gate_status": "pass",
    "gate_details": {
      "adherence_delta_pt": -1.2,

      "voice_shape_sigma": 0.3
    }
  },

  // Optional free-form annotation. Human or proposer-authored.
  "notes": "…"
}
```

## Gate thresholds (from the design doc)

Downstream replay passes when:
- `adherence_delta_pt` ≥ -5.0 (no more than 5pt regression)
- `halluc_delta_pt` ≤ +2.0 (no more than 2pt fire-rate increase)
- `voice_shape_sigma` ≤ 1.0 (within 1σ of pre-winner baseline)

Any one failing → `gate_status: "fail"`.

## Appending rules

- The file is append-only. Never rewrite a prior line.
- If a record was written in error, the next record's `notes` must
  call it out ("iter-003 config was a no-op — ignore in aggregation").
- No partial writes: one JSON object per line, newline-terminated,
  written atomically (`Bun.write` with the full payload).
