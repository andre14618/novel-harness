## 2. Architectural iterations with supersession chains

### Chain A: Invariant #5 detector precision

- **Initial approach (70f814d):** AST walk with receiver-DFS grouping + if-throw-return reachability classifier. 5/5 fixtures fire, HEAD clean.
- **Problem discovered:** Codex thread `a0b8a5d7` HIGH ‚Äî `resolveIdentifierDeclaration` was DFS-first-match over the function body, not scope-walk. Inner-block shadowed `const res = ...` would mis-group with an outer sibling declaration.
- **Superseded by (b5cb37a):** lexical scope walk ‚Äî walk UP from identifier through `isNewScope()` ancestors (Block / Function / For / Catch), check each scope's direct declarations, first match wins.
- **Problem re-surfaced:** Codex thread `a76243c1` MEDIUM ‚Äî my switch-terminator classifier called `blockEndsInThrowOrReturn` on each clause's last statement, but that helper counted `break` + `continue` as terminal. A switch whose clauses end in `break` falls through to the statement AFTER the switch ‚Äî not terminal for outer-scope reachability.
- **Superseded by (8cc3d2c):** stricter `exitsFunction()` helper (throw/return only + Block/If/Try recursion), used specifically in the switch-clause check.
- **Lesson:** Scope-local control transfers (`break`/`continue`) are not interchangeable with function-scope transfers (`throw`/`return`) for reachability analysis. The "terminal statement" check has to know its target scope.

## 4. Class-of-bug patterns

- **AST beats text for syntactic checks** ‚Äî elevated to `docs/patterns/ast-over-text-for-syntactic-invariants.md`. This session provided the second-session recurrence: invariant #5 moved from template-literal regex to AST receiver/path analysis in `70f814d`, then hardened in `b5cb37a` and `8cc3d2c`.
- **Scope-local vs function-scope control transfers** ‚Äî `break` / `continue` exit a switch or loop but not the enclosing function; `throw` / `return` do both. Reachability analysis must distinguish these. Seen 1x this session.
