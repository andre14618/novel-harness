## 2. Architectural iterations with supersession chains

- **What happened:** the Codex agent interpreted "triage" loosely and implemented the full ticket — pattern doc, back-links, commit-attempt. Sandbox blocked the commit; Claude committed manually as `b3a3195`.

- **Lesson:** the 13-phase pipeline is optimized for architectural charters. For routine tickets, the ceremony exceeds the value. Tier selection matters.

### Chain B: T4 classifier-semantics iteration

- **Initial plan:** extend `exitsFunction` to handle `while(true) { ...throw }`. Break/continue treated uniformly.
- **Codex triage (`a744b941`):** `red` — a `continue` inside a switch inside a loop still targets the enclosing loop; the helper's original capture rules would have missed this edge.
- **Fixed in plan:** keyword-specific capture rules — `break` captured by loop OR switch; `continue` captured by loop only.
- **Codex full review (`aef73a30`):** HIGH — `do { return } while (cond)` body runs unconditionally once; shouldn't require `isLiteralTrue(cond)`.
- **Fixed in plan:** do-while is terminal iff body exits, regardless of condition.
- **Implementation:** clean on first subagent pass. All 12 unit tests green including the do-while-with-condition case.
- **Lesson:** AST reachability is a field of edge cases. Exhaustive unit tests beat exhaustive plan writing — test each semantic claim instead of prose-describing it.

### Chain C: T7 plan-triage pedantry

- **Four triage rounds** before T7 plan hit green. Each round surfaced something real:
  - Round 1 (`a6aadc90`): `checker-eval` vs `checker_eval` naming mismatch.

## 4. Class-of-bug patterns

- **Scope-local vs function-scope control transfers** — recurring across T1 (switch-break), T4 (loop-break, loop-continue). `break` inside a switch is not `break` inside a loop is not `return`. Classifier can't treat these uniformly. Elevation candidate if it recurs one more time.
- **Widened-literal unions for loose-typed APIs** — `| (string & {})` tail preserves autocomplete while accepting ad-hoc strings. Applied in T7. Worth documenting if another API needs the same treatment.
- **Rogue agent boundary** — Codex subagents sometimes ignore strict contract prompts (`triage-only` → full implementation). Accept it when the outcome is good; codify as a new tier rather than fighting the behavior.
