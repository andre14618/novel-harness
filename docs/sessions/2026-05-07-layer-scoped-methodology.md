# Layer-Scoped Methodology

## Decision Capture

The user identified that trying to improve the whole harness at once creates
too many confounded variables. The next product work should narrow to upstream
planning methodology: structure templates, chapter contracts, scene functions,
and plan-quality diagnostics.

## Repository Rule

Every methodology change must name the optimized layer and keep adjacent
layers steady unless the experiment explicitly tests downstream projection.

Current priority:

- upstream concept/planning templates;
- chapter and scene story-function allocation;
- endpoint landing and character materiality;
- ID-linked planner-quality observations.

Deferred unless directly changed by a slice:

- extra UI/Playwright work;
- writer/reviser prompt changes;
- checker strictness changes;
- proposal/review UI expansion.

## Why

Scaffolds such as commercial chapter-function templates may help, but they can
also create unintended downstream effects if wired into drafting/checking too
early. Testing them as far upstream as possible lets the harness measure plan
quality before changing prose generation or review surfaces.
