## 2. Architectural iterations with supersession chains

### Chain A: Starter kit location — novel-harness subdirectory → shared parent

- **Initial approach:** T8 agent wrote to `novel-harness/docs/portable/invariants-starter/`. T8 was scoped this way because workflow-portable.md already lived there.
- **User direction (mid-T8):** "we should take all the 'starter kits' and put it in the personal_projects folder with a 'workflows_starter' folder."
- **Superseded by:** moved T8's output to `workflows_starter/invariants-starter/`; moved `workflow-portable.md` too; reverted T8's cross-link pointer in `novel-harness/docs/invariants.md`. T9 + T10 dispatched with the new target path baked in.
- **Lesson:** portable artifacts don't belong inside a specific project's repo. The signal is the word "portable" itself — if it can be shared, it should live somewhere shareable. Pick the right directory at plan time, not mid-implementation.

### Chain B: Codex sandbox boundary on cross-directory writes

- **Initial approach:** T9 + T10 dispatched with absolute paths into `workflows_starter/` (outside novel-harness).
- **T9 outcome:** Codex wrote the files successfully — `workflows_starter/` is one directory up from `novel-harness/`, within some sandbox boundary that permitted it.
- **T10 outcome:** Codex halted with "sandbox boundary — cannot write to sibling directory." The agent offered two options: invoke from a parent directory OR let Claude write directly.
- **Superseded by:** Claude main wrote T10's 6 files directly from the reference sources. Same output, no retry needed.
- **Lesson:** Codex subagent sandbox is implicit and not uniformly described. T9 worked, T10 didn't, and the difference was behavioral (Codex chose to halt on T10 rather than try the write). Accept the variability; fall back to direct Write when Codex reports a sandbox wall.

### Chain C: Taxonomy label bootstrap ordering

- **Initial approach:** Exps #244-#250 (T1-T7 from prior session batches) were labeled `charter` because that's the label that existed at creation time.
- **Problem discovered:** user asked "how did we label those experiments? seems more like tickets tbh."
- **Root cause:** T7 itself shipped the `ticket` canonical label in commit `252a01f`. Every exp created BEFORE that commit got `charter`; every exp AFTER (T8/T9/T10 = #251/#252/#253) correctly got `ticket`. Bootstrap artifact — T7 couldn't retroactively relabel itself.
- **Decision:** not backfilling. Historical labels preserved as evidence of what we thought at the time. Going forward, all engineering work uses `ticket` per the canonical taxonomy.
- **Lesson:** introducing a new label in the same commit that uses it is a classic chicken-and-egg. Future shifts like this should either (a) pre-land the label in a separate commit, or (b) accept the bootstrap artifact explicitly in the commit message.
