Single-page view of what's in flight, what's queued, and what's parked across the harness ‚Äî corpus mining, planner-prompt variants, writer-layer ablations, and checker work. **Living doc** ‚Äî update each row as work moves through the pipeline.

## How work moves through the pipeline

```
mine                ‚Üí  pattern / signal extracted from corpus or run output
map                 ‚Üí  pattern tied to a specific prompt / schema field / pipeline lever
draft variant       ‚Üí  the alternative phrasing or config that encodes the pattern
probe               ‚Üí  $0.30 phase-eval (planner) or full-novel sample (writer/checker)
cross-book / scale  ‚Üí  reproduces qualitatively in a 2nd book / scales beyond pilot N
ship / hold / pivot ‚Üí  variant lands, holds for more evidence, or kills
```

## Ship gate framing (load-bearing ‚Äî re-read before defining new gates)

The user's standing rule (2026-04-30): **‚â• 90% confident a pattern reproduces qualitatively is sufficient to ship as a planner prior.** Tight quantitative CIs are NOT required. Heavy statistical reads are diminishing-returns past book 2 or 3.

Math reference (binary proportions at p‚âà0.5):
- 1 book, n‚âà34 chapters ‚Üí 95% CI ¬±17% (range 33‚Äì67% on a 50% point estimate)
- 2 books ‚Üí ¬±12%
- 3 books ‚Üí ¬±10%
- 11+ books ‚Üí ¬±5%

Translation: cross-book validation is HIGH-VALUE for the binary "does the pattern reproduce" test (one additional book is enough to discriminate "Salvatore quirk" vs "author/genre pattern"). It's LOW-VALUE for tight quantitative priors. The current planner-prompt rules are already consistent with the wide n=34 CI; cross-book mostly answers reproduction, not precision. Don't gate variant ship decisions on CI tightness.

Anchor stability gates for stochastic-schema dims (separate from pattern-shipping): J ‚â• 0.85 Sonnet self-consistency at BOTH calibration-anchor AND production-emit granularities, validated at full population (n=50 is screening only). See `docs/decisions.md` 2026-04-30 SOPs.
