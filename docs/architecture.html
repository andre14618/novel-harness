2. Agent Call Flow

Phase function (e.g. runDraftingPhase)
      │
      ▼
callAgent(systemPrompt, userPrompt, schema)          src/llm.ts
      │
      ▼
getAgentConfig(agentName)                             models/roles.ts
      │  returns: provider, model, temp, maxTokens, thinking
      ▼
makeRequest()
      │
      ▼
getTransport().execute(request)
      │
      ├── LLM_TRANSPORT=batch ───▶ BatchTransport ──▶ queue ──▶ flush to /v1/batches
      │
      └── default ───▶ DirectTransport
                            │
                            ▼
                   HTTP POST to provider
                   retry 3x on 429/5xx
                            │
                            ▼
                   parse JSON + Zod validate
                            │
                            ▼
                   logLLMCall ──▶ Postgres
                   (provider prefix caching happens automatically — no transport logic needed)

3. Two-Database Architecture

PER-NOVEL SQLITE  output/{novelId}/novel.db  (bun:sqlite, one per novel run)
┌─────────────────────────────────────────────────────────────────────┐
│  novels ─── world_bibles ─── characters ─── story_spines           │
│  chapter_outlines ─── chapter_drafts ─── chapter_summaries         │
│  facts ─── character_states ─── issues ─── validation_passes       │
│                                                                     │
│  Creative content for a single novel run                          │
│  Created at novel start, read during drafting/validation           │
└─────────────────────────────────────────────────────────────────────┘

CENTRAL POSTGRES  novel_harness_orchestrator  (LXC 307)
┌─────────────────────────────────────────────────────────────────────┐
│  TRACKING           BENCHMARKS          IMPROVEMENT     BATCH      │
│  ─────────          ──────────          ───────────     ─────      │
│  runs               generations         tuning_exps    orch_       │
│  run_agents         scores              improvement_    batches    │
│  llm_calls          baselines             cycles       orch_       │
│                     lint_patterns       improvement_    requests   │
│                     lint_issues           iterations               │
│                     pairwise_matchups   budget_tracker             │
│                                                                     │
│  All operational data across all runs. Never deleted.              │
└─────────────────────────────────────────────────────────────────────┘

Data flow:
  Novel agents ──write──▶ SQLite  (creative content)
  Novel agents ──log────▶ Postgres (llm_calls, runs)
  Benchmarks ───────────▶ Postgres (generations, scores)
  Improvement daemon ───▶ Postgres (cycles, iterations, experiments)
  Batch jobs ───────────▶ Postgres (batches, requests)
