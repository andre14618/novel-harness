## L31d-(NEW): Continuity blocker → immediate plan-assist gate

**Mechanism (`src/phases/drafting.ts:1116-1133`):**

```typescript
const checkerBlockers = buildCheckerBlockerDeviations({
  acceptedBeatIssues, continuityIssues: issues, functionalIssues,
})
if (checkerBlockers.length > 0) {
  // sets pendingExhaustion (kind: "plan-check-exhausted")
  // bail = true
}
```

`buildCheckerBlockerDeviations` (`src/phases/checker-blockers.ts`) collects every `severity: "blocker"` continuity issue and constructs a `chapter-level` deviation with no beat_index. The chapter-attempt loop then bails immediately and routes to the plan-assist gate. There is no chapter-attempt retry for continuity blockers — they go straight to operator review (or auto-mode failure).

This is the documented design: continuity is the FINAL gate after beat-level rewrites and plan-check rewrites have already settled. The expectation is that the writer/beat-writer chain produces correct state propagation, and continuity catches the rare planner→prose divergence that escaped beat-level checking.

**Why it fires here:** the planner specified `obligation: ledger-glows-red-near-false-debts` (set in chapter 1). Beat 1 of chapter 2 has Taryn entering the Sorcerer's Guild Library hidden alcove containing the false-debt records. The writer rendered the ledger as glowing amber and being "quiet" — likely because the prose described the alcove ambience first and the ledger followed. A targeted beat-level rewrite would likely fix this.

**Recommendation:** L37 (or equivalent next-loop): either (i) include continuity blockers in the chapter-attempt retry budget — at minimum allow ONE attempt to invoke the chapter-plan-reviser with the continuity description, or (ii) accept current routing to plan-assist as correct and document this in `docs/overnight-runbook.md` so operators recognize "continuity blocker on attempt 1" as a normal halt requiring a manual outline edit or beat-level correction. A deeper investigation should also probe why the writer rendered the ledger as amber/quiet when the obligation chain established red-near-false-debts — likely the world-bible rule about the ledger's color is not propagating into beat-level prompts.
